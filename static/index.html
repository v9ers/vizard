Vizard
JSON goes in - graphs come out
(you can't explain that)
Vizard is an easy way to visualize your data - in realtime.

Send JSON data to the server, open up the corresponding display page and see your data. The page is updated in realtime when new data is posted.

A great alternative to building your own complicated infrastructure to visualize the data you have. Simply post it to the server and see it.

Examples

Irene on Twitter - The number of tweets with the hashtag #irene. Accumulated for every 10 minutes

Trending hashtags - See the most trending hashtags on Twitter over the last 10 minutes.

Fake movie data - An example about fake data being posted to the server.

Installation guide

1. Try it out online

First post some data. You can do this with curl.
%> curl -X POST --data '{"foo":"bar", "somenumber":42}' http://v9-ers.nko2.nodeknockout.com/r/nodeknockout
Instead of nodeknockout you can write something else. Use your imagination.
Now open this page in the browser
http://v9-ers.nko2.nodeknockout.com/nodeknockout
Obviously... this should match whatever name you chose before.
That's it! You see the first data online. Try to post some more data, change numbers, and generally play around with it.

2. Download the source

As anyone else we were forced to use Github. Feel free to fork the project if you want to.
Git repository

3. Run locally

First start the server. It will open on
%> node server.js
Trending hashtags - See the most trending hashtags on Twitter over the last 10 minutes.

Open an example page, and follow the tutorial
Open localhost:8008/nodeknockout

Notes

One of the best examples of Vizard is to use it on your own server with internal data. It's a great way to easily visualize data without having to built a big infrastructure.

In the weekend we have built this we have used it to visualize our own dat on a service we run, and it helped us to easily identify a few pitfalls. Just by sending the JSON data we had to the server.
